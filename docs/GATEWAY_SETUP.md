# اتصال اپ به Gateway — اولین بار

این **مرحله‌ی ۳ از ۳** برای راه‌اندازی Hermes2 است. پیش‌نیاز: [نصب هرمس](INSTALL_HERMES_TERMUX.md) و [ویزارد راه‌اندازی](SETUP_HERMES_TERMUX.md) را تمام کرده باشی.

مراحل این صفحه را فقط **یک‌بار**، برای اولین اتصال انجام می‌دهی. بعد از آن، هر بار که اپ را باز کنی **خودکار وصل می‌شود** و دیگر لازمشان نداری.

---

## مرحله ۰ — نصب اپ Hermes2

اگر هنوز اپ را نصب نکرده‌ای:

۱. آخرین APK را از **[صفحه‌ی Releases](https://github.com/traveler3022/Hermes2/releases/latest)** دانلود کن
۲. فایل APK را باز کن و نصب را تأیید کن (اندروید ممکن است اجازه‌ی «نصب از منابع ناشناس» بخواهد)

---

## مرحله ۱ — آزادکردن پورت‌های Termux

داخل Termux این دستور را بزن تا اگر dashboard دستی در حال اجراست، متوقف شود:

</div>

```bash
hermes dashboard --stop
```

<div dir="rtl">

حالا **از Termux خارج شو** و آن را force-stop کن:

**تنظیمات اندروید ← برنامه‌ها ← Termux ← Force stop**

> **چرا force-stop؟** موقع نصب، Termux پورت‌هایی را اشغال می‌کند و رها نمی‌کند. با force-stop تمام پورت‌های باز آزاد می‌شوند تا اپ بتواند یک اتصال تمیز برقرار کند. این فقط **دفعه‌ی اول** لازم است.

---

## مرحله ۲ — اتصال از داخل اپ

۱. اپ **Hermes2** را باز کن
۲. روی **`Open runtime host app`** بزن ← Termux باز می‌شود
۳. **برگرد به اپ Hermes2**
۴. روی **`Start Agent Gateway`** بزن
۵. **تا ۳۰ ثانیه صبر کن** — وضعیت به **✓ Connected** تغییر می‌کند

> **چرا از اپ استارت کنیم، نه دستی؟**
> اپ خودش یک `HERMES_DASHBOARD_SESSION_TOKEN` تولید و تزریق می‌کند. اگر دستی `hermes dashboard` را با توکن دیگری استارت کنی، احراز هویت WebSocket شکست می‌خورد.

> [!TIP]
> اگر تا ۳۰ ثانیه وصل نشد، **مراحل ۱ و ۲ را یک‌بار دیگر تکرار کن.** اولین دست‌دادن (handshake) گاهی بار دوم موفق می‌شود — بعد از آن همیشه خودکار وصل می‌شود.

---

## از دفعه‌ی دوم به بعد

فقط اپ را باز کن؛ خودش وصل می‌شود. نه Termux لازم است، نه force-stop، نه هیچ کار اضافی.

---

## بعد از اتصال

به **Settings** اپ برو و چک کن:

</div>

```text
Provider: gemini (یا هر ارائه‌دهنده‌ای که انتخاب کردی)
Model: gemini-2.5-flash یا gemini-2.5-pro
```

<div dir="rtl">

تب **Tools** هم باید ابزارهای فعال هرمس را نشان بدهد.

---

## اگر با خاموش‌شدن صفحه اتصال قطع شد

- **تنظیمات ← برنامه‌ها ← Hermes2 ← باتری ← بدون محدودیت**
- همین کار را برای **Termux** هم انجام بده

---

## عیب‌یابی

### لاگ gateway

</div>

```bash
cat "$HOME/.hermes/logs/gateway_stdout.log"
```

<div dir="rtl">

یا از داخل اپ: **Termux & Agent Connection ← Fetch & View Logs**

### از کجا بفهمم هرمس خودش سالم است؟

در Termux:

</div>

```bash
hermes doctor
```

<div dir="rtl">

باید `✓ gemini (key configured)` (یا provider خودت) را ببینی. اگر doctor خطا داد، مشکل از هرمس است نه اپ — [راهنمای نصب](INSTALL_HERMES_TERMUX.md#عیبیابی) را ببین.
